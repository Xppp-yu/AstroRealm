## 知识要点

### Hermes Agent 分析
- GitHub: https://github.com/nousresearch/hermes-agent.git
- 功能：独立的 AI 代理框架，与 Claude Code 类似但独立
- 评估：按现有架构不需要安装，Claude Code 已满足需求

### Lark/Feishu CLI 核心功能
| 模块 | 能力 | AI 友好度 |
|------|------|-----------|
| Calendar | 日程创建/读取/更新/删除 | ⭐⭐⭐⭐⭐ |

| IM | 消息发送/群聊管理 | ⭐⭐⭐⭐⭐ |

| Sheets | 表格数据操作 | ⭐⭐⭐⭐ |
| Mail | 邮件收发 | ⭐⭐⭐⭐ |

### 安装配置流程
```bash
# 1. 安装 CLI 工具
npm install -g @larksuite/cli

# 2. 安装 Claude Skill
npx skills add larksuite/cli -y -g

# 3. 配置应用 (需要浏览器操作)
lark-cli config init --new

# 4. OAuth 授权 (需要浏览器操作)
lark-cli auth login --recommend
```

### 安全授权机制
- **Skill 安装 ≠ 直接访问**：Skill 只提供命令知识
- **OAuth 授权必需**：用户必须在浏览器中完成授权
- **权限粒度可控**：可以精确控制授予哪些 API 权限
- **凭证本地存储**：Token 存储在操作系统密钥链中

### AI 代理操作模式
| 模式 | 描述 | 用户控制度 |
|------|------|-----------|
| **建议模式** | AI 提供命令，用户手动执行 | 100% 控制 |
| **执行模式** | AI 通过 Bash 工具直接执行 | 需要额外授权 |
| **混合模式** | 关键操作用户确认，重复操作 AI 执行 | 平衡效率与控制 |
