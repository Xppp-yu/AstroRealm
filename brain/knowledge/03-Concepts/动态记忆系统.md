# 动态记忆系统

> 跨模型一致性协作系统 - 让任何 API 都能保持一致的协作体验

## 概述

这个系统解决的核心问题：**更换模型 API 后，如何保持一致的协作体验？**

## 核心理念

1. **动态迭代** - 人设和协作模式根据反馈实时优化
2. **模型无关** - 任何模型 API 都能获得完整上下文
3. **双库协同** - CoBrain 记录知识，ClaudeStudio 执行项目

## 系统架构

```
│  ────────────────                                           │
│  - pending.md：待处理反馈                                    │
```

## 工作流

```
会话启动                     会话结束
    │                           │
    ▼                           ▼
┌──────────────┐         ┌──────────────┐
│ 加载动态人设  │         │ 生成会话摘要  │
│ 加载知识库    │  ───▶   │ 检查反馈队列  │
│ 加载上次摘要  │         │ 触发迭代检查  │
└──────────────┘         └──────────────┘
                               │
                               ▼
                         ┌──────────────┐
                         │ 迭代优化脚本  │
                         │ 更新人设版本  │
                         │ 同步到 CoBrain│
                         └──────────────┘
```

## 如何使用

### 日常使用
1. 正常启动 Claude Code，系统自动加载人设和知识库
2. 协作过程中，如有反馈，编辑 `feedback/pending.md`
3. 会话结束时，系统会提示是否有待处理反馈

### 提供反馈
```markdown
### [日期] 简短描述

**场景**：这个反馈发生在什么情况下
**问题**：具体是什么不满意
**期望**：希望怎么改进
**优先级**：高/中/低
```

### 触发迭代
```bash
./hooks/iterate.sh
```

