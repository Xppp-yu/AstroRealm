# 决策：ClaudeStudio Git 管理策略

## 背景
ClaudeStudio 作为工作目录，需要决定 Git 管理策略。

## 备选方案
| 方案 | 优点 | 缺点 |
|------|------|------|
| A. 一个大仓库 | 管理简单 | 推送时全部项目一起 |
| B. 每个项目独立仓库 | 灵活分享 | 管理复杂 |

## 最终决策
采用**混合策略**：
- ClaudeStudio 整体作为一个 Git 仓库（本地版本控制）
- 未来做得好的项目可单独分离推送

## 决策理由
1. 本地版本控制方便回溯
2. 灵活性：随时可以单独推送项目
3. 管理简单：不需要每个项目都 init

## 安全措施
- 使用 `git checkout <commit> -- <目录>/` 只恢复特定项目
- 使用分支隔离实验性修改
