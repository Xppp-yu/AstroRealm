# 协作模式库

> 记录成功的协作模式，供未来参考

## 模式 1：知识库驱动协作

**场景**：需要基于用户已有知识进行工作

**流程**：
1. SessionStart Hook 自动加载 CoBrain 知识库状态
2. AI 获得完整的知识上下文
3. 基于知识库内容进行协作
4. 新知识自动记录到 CoBrain

**效果**：更换 API 后仍能保持知识连续性

---

## 模式 2：三层记忆体系

**场景**：需要保持跨会话的上下文

**流程**：
1. **长期记忆** - 文件存储，Hook 注入
2. **中期记忆** - 会话摘要，下次加载
3. **短期记忆** - 当前会话上下文

**效果**：任何模型启动时都获得完整上下文

---

## 模式 3：动态人设迭代

**场景**：协作风格需要不断优化

**流程**：
1. 收集用户反馈到 `feedback/pending.md`
2. 定期分析反馈
3. 更新 `persona/preferences.md`
4. 保存版本历史

**效果**：人设随使用而进化

---

## 模式 4：双库协同

**场景**：项目执行 + 知识记录

**流程**：
```
ClaudeStudio (执行) ←→ CoBrain (记录)
     ↓                      ↓
  项目文件               知识卡片
     ↓                      ↓
  GitHub                 本地库
```

**效果**：执行和记录分离，各自优化
